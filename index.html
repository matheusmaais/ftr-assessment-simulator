Prévia de roteamento

O mesmo core de assessment, duas formas diferentes de evidência.

Partner Hosted

Maior potencial de automação futura: APIs AWS, CIS, Security Hub, configuração e snapshots.

Service Offering

Maior valor em workflow assistido: SoW, processo, governança, acesso e feedback do cliente.

O que validar com o cliente

  • Qual tipo de FTR representa melhor a solução ou oferta avaliada.
  • Quais evidências o cliente já tem e quais ainda precisam ser produzidas.
  • Onde o processo trava hoje: scan, escrita, diagrama, evidência ou Partner Central.
  • Quais gaps parecem mais críticos antes de envolver reviewer ou PSA.

O que é este assessment fictício?

É um caso inventado de uma plataforma SaaS hospedada na AWS. Ele mostra como a ferramenta organizaria contexto, evidências recebidas, hipóteses, gaps e findings.

  • Não veio de scan real e não acessa nenhuma conta AWS.
  • Serve para demonstrar o formato de conversa e saída do assessment.
  • Cliente fictício: CloudBilling Pro, SaaS B2B Partner Hosted.

Assessment automatizado mockado de conta AWS

Este exemplo simula o resultado de um scan read-only em uma conta AWS fictícia. Ele mostra inventário, fontes de evidência e findings técnicos como se a ferramenta tivesse coletado snapshots.

  • Não executa AWS API real e não usa credenciais.
  • Conta fictícia: 123456789012, regiões us-east-1 e sa-east-1.
  • Inclui findings mockados de S3, CloudTrail, Security Hub, IAM, KMS e backup.
